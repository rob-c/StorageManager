Built for physicists and students who want their files, not a tutorial on SSHFS. Type a password, pick a folder, get a drive.

What it does

One tool for the whole remote-storage chore.

Everything you'd otherwise cobble together from sshfs flags, ssh_config edits, and half-remembered kinit commands — in one window.

Mount

One-click drives

Enter your username and password, pick a folder, and it appears as a drive — S: on Windows, ~/S on macOS and Linux. Read-only by default, so nothing changes by accident.

Fewer tickets

Knows when you're off-VPN

Checks the server is reachable before mounting and, if it isn't, tells you to connect to the university VPN — instead of a cryptic timeout you have to google.

SSH Doctor

Fixes your ssh_config

Audits ~/.ssh/config for missing keepalives, broken jump hosts, DPI-hostile options, and security foot-guns — then applies the fixes with a backup first.

VS Code

Remote dev that just works

Installs the Remote-SSH extension and writes the right Host block — jump hosts included — so "Connect to Host" and 2FA prompts work the first time.

Status

Kerberos & quota, together

Grab a CERN ticket with one click and see your AFS and EOS usage against quota in one place — the ticket lets the quota checks run without another password.

Anywhere

Window, terminal, or script

A friendly desktop window, a guided terminal UI for headless SSH sessions, and scriptable --doctor / --status commands — the same engine, three ways.

How it works

Three steps from launch to files.

Sign in

Type your university username and password. Storage Manager checks the server is reachable first, so you fail fast with a clear message rather than a hang.

Mount

Pick a host and folder — Edinburgh datastores and CERN AFS/EOS are built in. It mounts read-only as a drive; unreadable errors get translated into plain English.

Work

Open it in your file manager or in VS Code. Need to change files? Tick "Allow read-write" and the status line shows you're now writable.

Not just a GUI

Scriptable, and honest about what's wrong.

The SSH Doctor runs anywhere — in the window, in the terminal UI, or straight from a shell. It reports exactly what it found and offers a fix you can preview as a diff before it touches a thing.

$ StorageManager --doctor lxplus --fix

StorageManager --doctor 
SSH Doctor — host 'lxplus' (~/.ssh/config)

[ERROR]   Jump host 'gateway' is not configured
          ProxyJump references 'gateway', but no Host block
          defines it. ssh will fail to resolve the jump.

[WARNING] No ServerAliveInterval set
          Idle sessions get dropped with no warning.
          fix → Set ServerAliveInterval 30 for lxplus

[WARNING] StrictHostKeyChecking is disabled
          fix → Set StrictHostKeyChecking accept-new

Applied 2 fixes. Backup: ~/.ssh/config.bak-20260712
